## 📌 Introduction
The **Student Marketplace Platform** is a web-based application inspired by platforms like NoteKart, designed to enable students to buy, sell, and exchange academic resources such as notes, books, electronics, and study materials.

This project is enhanced with **DevOps practices**, enabling automated deployment, scalability, and continuous integration for real-world production environments.

---

## 🎯 Objectives
- Build a student-centric marketplace platform  
- Enable peer-to-peer exchange of academic resources  
- Implement **DevOps pipeline for automation and scalability**  
- Ensure high availability, performance, and reliability  

---

## ❗ Problem Statement
Students face:
- High cost of academic materials  
- Lack of trusted platforms  
- Difficulty in managing buying/selling  

Traditional systems lack:
- Automation  
- Scalability  
- Efficient deployment processes  

---

## 💡 Proposed Solution
The platform provides:
- A centralized marketplace for students  
- Secure login and product listing  
- Search and communication features  
- Automated deployment using DevOps tools

### 🔹 Architecture Description
The system follows a **3-tier architecture with DevOps pipeline**:

- **Frontend:** React.js  
- **Backend:** Node.js + Express  
- **Database:** MongoDB  

DevOps Integration:
- GitHub for version control  
- Jenkins/GitHub Actions for CI/CD  
- Docker for containerization  
- AWS for deployment  
- Nginx for reverse proxy  
- Prometheus & Grafana for monitoring

## 🔄 CI/CD Workflow

  <img src="docs/workflow.png" width="800"/>
</p>

### 🔹 Workflow Steps
1. Developer writes code  
2. Code is pushed to GitHub repository  
3. CI/CD pipeline is triggered  
4. Application is built and tested automatically  
5. Docker image is created  
6. Application is deployed on AWS  
7. Nginx serves the application  
8. Monitoring tools track performance
